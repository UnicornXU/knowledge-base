# 🎨 前端可视化

> **"一图胜千言，一帧定体验"** —— 前端可视化是将数据与逻辑转化为直观图形的技术，是高级前端工程师的核心技能之一。

## 为什么需要学习前端可视化？

```
数据 → 可视化 → 洞察
────────────────────────────────────────────────────

  原始数据（表格、JSON）
        ↓
  ┌─────────────────────────────┐
  │  可视化引擎                  │
  │  ├── Canvas 2D / WebGL      │
  │  ├── SVG                    │
  │  ├── ECharts / D3.js        │
  │  └── CSS / HTML             │
  └─────────────────────────────┘
        ↓
  图表、地图、大屏、3D 场景
        ↓
  用户快速获取信息、辅助决策
```

**面试中可视化领域的考察越来越频繁**，因为它综合考察了渲染原理、性能优化、算法能力等多维度技能。

## 知识图谱

```
前端可视化
├── 🖼️ Canvas           ── 2D 绘图 API、WebGL、离屏 Canvas、动画循环
├── ✏️ SVG              ── 矢量图形、viewBox、动画、与 Canvas 对比
├── 📊 ECharts          ── 配置体系、自定义系列、性能优化、按需加载
├── ⚡ 大数据量渲染      ── 虚拟列表、Canvas 分层、Web Worker、WebGL 点渲染
└── 🔧 工程实践          ── 图表选型、响应式适配、无障碍、服务端渲染
```

## 技术选型指南

```
如何选择渲染技术？
────────────────────────────────────────────────────

  数据量 < 1000 个元素？
  ├── 是 → SVG（交互方便、DOM 事件）
  └── 否 ↓

  需要像素级操作或 3D？
  ├── 是 → WebGL
  └── 否 ↓

  数据量 1000 ~ 100000？
  ├── 是 → Canvas 2D（离屏渲染 + 分层优化）
  └── 否 ↓

  数据量 > 100000？
  └── WebGL + 数据抽样 + Worker 预处理
```

## 渲染技术对比

| 特性 | Canvas 2D | WebGL | SVG | CSS/HTML |
|------|-----------|-------|-----|----------|
| 渲染方式 | 位图（像素） | 位图（GPU 加速） | 矢量（DOM） | DOM |
| 交互性 | 需手动实现 | 需手动实现 | 原生 DOM 事件 | 原生事件 |
| 大数据量 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐ | ⭐ |
| 动画性能 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐ |
| 可访问性 | 差 | 差 | 好 | 好 |
| 适用场景 | 图表、游戏 | 3D、粒子、大规模数据 | 图标、地图、交互图表 | 简单图表 |

## 学习建议

1. **Canvas 是基础**：理解 2D 绘图 API 和动画循环，是所有 Canvas 可视化的前提
2. **SVG 不可忽视**：矢量图形在图标、地图、交互式图表中有独特优势
3. **ECharts 是实战利器**：大部分中后台项目的图表需求都可以用 ECharts 解决
4. **大数据渲染是进阶**：掌握虚拟列表、分层渲染、Web Worker 等优化手段
5. **WebGL 是高阶方向**：3D 可视化、大规模数据渲染的终极方案
