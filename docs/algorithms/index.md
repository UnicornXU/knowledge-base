# 🧮 数据结构与算法

> **"算法是程序的灵魂"** —— 数据结构与算法是计算机科学的基石，也是前端面试中区分工程师深度的关键指标。

## 为什么前端也需要学算法？

```
你以为前端只是写页面？
    ↓
实际上你需要：
┌──────────────────────────────────────────────────┐
│  虚拟 DOM Diff → 树的遍历与比较算法               │
│  状态管理     → 图的依赖追踪                      │
│  性能优化     → 节流/防抖 → 函数式编程思想         │
│  数据处理     → 排序/搜索/去重 → 数组操作          │
│  可视化/游戏  → 空间索引、碰撞检测、路径规划        │
│  工程化       → 依赖拓扑排序、AST 遍历             │
└──────────────────────────────────────────────────┘
    ↓
算法能力决定了你解决问题的深度和广度
```

**大厂面试中，算法题是必考环节**，尤其是字节、美团、阿里等公司，手写算法题的比重越来越高。

## 学习建议

1. **先理解数据结构** —— 数据结构是算法的载体，不理解数据结构就无法理解算法
2. **从简单排序开始** —— 冒泡、选择、插入排序帮助建立复杂度直觉
3. **重点掌握快排和归并** —— 面试高频，也是理解分治思想的入口
4. **动态规划需要大量练习** —— 从斐波那契开始，逐步到背包、子序列问题
5. **结合前端场景** —— 理解虚拟 DOM Diff、AST 遍历等实际应用

## 面试要点

> 算法面试通常占 **30-40 分钟**，需要在白板或在线编辑器上完成。建议每天刷 1-2 题，保持手感。重点掌握：双指针、滑动窗口、BFS/DFS、动态规划这四大类题型。
