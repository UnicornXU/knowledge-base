# 低代码平台

低代码平台通过可视化拖拽、配置化表单和模板化页面搭建，大幅降低应用开发门槛。掌握低代码核心原理是前端高级工程师的重要能力。

## 知识图谱

```
低代码平台
├── 拖拽引擎
│   ├── HTML5 Drag & Drop API
│   ├── Sortable.js 排序原理
│   ├── 拖拽排版与对齐辅助线
│   └── 组件物料与拖拽注册
├── JSON Schema 表单引擎
│   ├── Schema 定义规范（JSON Schema / 自定义协议）
│   ├── 表单渲染引擎（递归渲染）
│   ├── 校验联动与动态显隐
│   └── 自定义组件扩展
├── H5 编辑器架构
│   ├── 画布设计（Canvas / DOM 双模式）
│   ├── 组件树与状态管理
│   ├── 属性面板（Property Panel）
│   └── 预览与导出（JSON DSL → 页面）
└── 低代码平台全景
    ├── 物料体系（组件市场）
    ├── 数据源与 API 编排
    ├── 权限与多租户
    └── 插件化与可扩展架构
```

```
拖拽引擎（核心） → JSON Schema 表单 → H5 编辑器架构 → 低代码全景
   (基础能力)         (配置化核心)       (产品形态)       (平台视野)
```

## 面试考察重点

1. **拖拽引擎**：HTML5 Drag API 的局限性与兼容方案，Sortable.js 的 DOM diff 策略
2. **表单引擎**：JSON Schema 如何驱动表单渲染，校验联动的实现方式
3. **编辑器架构**：DSL 设计、组件树状态管理、画布渲染模式的选择
4. **工程化**：低代码平台的插件化设计、物料体系、版本管理

### 低代码 vs 传统开发

| 维度 | 传统开发 | 低代码平台 |
|------|----------|------------|
| 开发效率 | 逐行编码 | 拖拽 + 配置 |
| 技术门槛 | 需要编程基础 | 业务人员可上手 |
| 灵活性 | 完全自由 | 受组件和配置能力限制 |
| 可维护性 | 代码版本管理 | DSL 版本管理 |
| 适用场景 | 复杂业务逻辑 | CRUD、表单、审批流、营销页面 |
| 扩展性 | 无限扩展 | 需要插件或自定义组件 |

### 适用场景

- **企业内部工具**：审批流程、数据报表、配置管理
- **营销活动页面**：H5 活动页、表单收集、问卷调查
- **数据看板**：Dashboard、数据大屏
- **低代码表单**：问卷、审批表、数据录入
