# ♿ 前端无障碍访问（Accessibility）

> **"Web is for everyone"** —— Tim Berners-Lee，万维网之父

无障碍访问（Accessibility，简称 a11y）是指让所有人——包括残障人士——都能平等地获取和使用 Web 内容。这不仅是一项技术要求，更是一种社会责任。

## 为什么无障碍如此重要？

```
┌──────────────────────────────────────────────────────────┐
│                   全球残障人口数据                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  🌍 全球超过 10 亿人有某种形式的残障（WHO 数据）            │
│  👁️ 约 2.53 亿人有视力障碍                                │
│  👂 约 4.66 亿人有听力障碍                                 │
│  🧠 约 2 亿人有认知障碍                                    │
│  🖐️ 数亿人有运动障碍                                      │
│                                                          │
│  💡 老年用户、临时性损伤（如手臂骨折）也会受益于无障碍设计    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**商业价值：**
- 可访问的网站 SEO 表现更好（语义化 HTML 天然对搜索引擎友好）
- 扩大用户群体，覆盖更多潜在客户
- 避免法律风险（ADA、EN 301 549 等法规）
- 提升整体用户体验（"路缘效应"——无障碍设计惠及所有人）

## WCAG 2.1 核心原则

Web 内容无障碍指南（WCAG）是国际公认的无障碍标准，其核心原则可概括为 **POUR**：

| 原则 | 英文 | 含义 | 示例 |
|------|------|------|------|
| **可感知** | Perceivable | 信息必须以用户能感知的方式呈现 | 替代文本、字幕、色彩对比度 |
| **可操作** | Operable | 界面组件必须可操作 | 键盘导航、足够的点击区域、无闪烁 |
| **可理解** | Understandable | 信息和操作必须可理解 | 清晰的标签、一致的导航、错误提示 |
| **健壮性** | Robust | 内容必须足够健壮，兼容辅助技术 | 语义化 HTML、标准 ARIA |

### WCAG 2.1 合规等级

| 等级 | 要求 | 适用场景 |
|------|------|----------|
| **A** | 最基本的无障碍要求 | 所有网站必须满足 |
| **AA** | 中等要求，大多数法规要求的等级 | 政府、企业网站推荐 |
| **AAA** | 最高等级，最严格 | 特殊场景（如医疗、教育） |

## 面试高频问题

1. **什么是 WCAG？其核心原则是什么？** —— 回答 POUR 四原则
2. **ARIA 的作用是什么？什么时候需要用 ARIA？** —— 补充语义，HTML 优先原则
3. **如何实现键盘可访问的自定义组件？** —— Focus Trap、Roving TabIndex
4. **如何测试网页的无障碍性？** —— 自动化工具 + 屏幕阅读器 + 键盘测试
5. **`aria-hidden` 和 `display: none` 有什么区别？** —— 对辅助技术的可见性差异

## 学习建议

1. **先理解原则**：掌握 WCAG 的 POUR 原则是所有无障碍工作的基础。
2. **语义化优先**：能用原生 HTML 元素就不用 ARIA，"No ARIA is better than bad ARIA"。
3. **动手测试**：关闭显示器，用键盘和屏幕阅读器体验自己的网站。
4. **渐进增强**：从 A 级开始，逐步达到 AA 级合规。
