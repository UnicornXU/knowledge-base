# 什么是 AI Agent

## 一个通俗的比喻

想象你雇了一位**智能私人助理**。你跟他说"帮我查一下明天北京到上海的高铁票，选一趟上午出发的，订最便宜的那趟"。一个普通的聊天机器人会告诉你"你可以去 12306 查"或者编一个答案；而一个真正的助理会：

1. **自己理解**你的需求（明天、北京到上海、上午、最便宜）
2. **自己规划**步骤（先查车次 → 筛选上午 → 比较价格 → 下单）
3. **自己动手**去查票系统查、去比价、去下单
4. **记住**你之前的偏好（比如你习惯坐靠窗）
5. 中间如果没票了，**自己调整**方案（换一趟或换日期）

这位"能自己思考、自己动手、能记住事、遇到问题会变通"的助理，就是 **AI Agent**。

> **一句话定义**：AI Agent 是一个以 LLM（大语言模型）为大脑，能够**自主感知、规划、决策、使用工具、记忆上下文**，从而完成复杂任务的智能系统。

## Agent vs 普通聊天机器人

很多人分不清 Agent 和 Chatbot。它们的本质区别在于：**Chatbot 只会"说"，Agent 还会"做"**。

| 对比维度 | 普通聊天机器人（Chatbot） | AI Agent |
|---------|------------------------|----------|
| **交互模式** | 一问一答，单轮或简单多轮 | 给一个目标，自己拆解并执行多步 |
| **能否使用工具** | ❌ 不能，只能基于训练知识回答 | ✅ 能调用搜索、计算、API、数据库等 |
| **自主性** | 低，完全由用户驱动 | 高，能自己决定下一步做什么 |
| **记忆能力** | 通常只有当前对话窗口 | 短期+长期记忆，能跨会话记住信息 |
| **纠错能力** | 答错了就答错了 | 能观察结果、反思、重新规划 |
| **典型例子** | 早期的 Siri、网站客服机器人 | AutoGPT、Manus、Devin、Cursor Agent |
| **输出形态** | 文字回答 | 文字 + 实际动作（发邮件、写文件、订票） |

:::tip 关键区别
Chatbot 是"你问我答"的**应答者**；Agent 是"你定目标，我来办"的**执行者**。
:::

一个更直观的对比：

```
用户："帮我查下明天北京天气"

Chatbot： "我无法访问实时数据，建议你查看天气预报 App。"
         （它没有工具，只能说"做不到"）

Agent：   [思考] 需要调用天气查询工具
         [行动] 调用 getWeather("北京", "明天")
         [观察] 工具返回：晴，最高 28°C
         [回答] "明天北京天气晴朗，最高气温 28°C，适合出行。"
         （它真的去查了，并基于真实数据回答）
```

## Agent 的四大核心特征

一个真正的 Agent，通常具备以下四个核心能力。我们用"助理"的比喻来理解每一个：

### 1. 自主性（Autonomy）

普通程序是"你写好流程，它照着走"；Agent 是"你给目标，它自己找路走"。

> 比喻：你给导航软件输入终点，它自己规划路线；遇到堵车还能重新规划。Agent 就是给 LLM 一个目标，让它自己决定中间每一步。

```typescript
// 普通程序：流程是写死的
function fixedFlow() {
  const data = fetch('/api/data');
  const result = process(data);
  save(result);
}

// Agent：流程是模型自己决定的
async function agentFlow(goal: string) {
  // 模型自己决定：先做什么、再做什么、用什么工具
  const plan = await llm.plan(goal);
  for (const step of plan.steps) {
    const tool = pickTool(step); // 自主选择工具
    const result = await tool.run(step.params);
    // 自主判断：结果对不对？要不要调整？
    if (!result.ok) await replan();
  }
}
```

### 2. 工具使用（Tool Use）

这是 Agent "能做事"的关键。LLM 本身只会生成文字，但通过 **Function Calling**，Agent 可以调用外部工具：搜索引擎、计算器、数据库、发邮件、操作文件……

> 比喻：LLM 是一个博学但"手无缚鸡之力"的顾问，工具就是它的"手和脚"。给它装上"手"，它就能干活。

### 3. 规划能力（Planning）

面对复杂任务，Agent 能把大目标拆成小步骤。比如"帮我做一份竞品分析报告"会被拆成：收集竞品信息 → 对比功能 → 分析优劣势 → 生成报告。

> 比喻：就像项目经理把一个大项目拆成一个个可执行的子任务，分配下去逐个完成。

### 4. 记忆能力（Memory）

Agent 能记住对话历史（短期记忆），也能把重要信息存到向量数据库里长期记住（长期记忆），还能在执行任务时临时记住中间结果（工作记忆）。

> 比喻：短期记忆像"便签纸"，长期记忆像"笔记本"，工作记忆像"手边的草稿纸"。

## Agent 的典型应用场景

| 场景 | Agent 在做什么 | 为什么需要 Agent |
|------|---------------|----------------|
| **编程助手**（Cursor/Cline） | 读代码 → 定位 bug → 改代码 → 跑测试 → 修错 | 需要多步操作文件、执行命令、验证结果 |
| **信息调研** | 搜索 → 提取 → 汇总 → 生成报告 | 需要多次搜索、整合多个来源 |
| **客服自动化** | 查订单 → 查物流 → 申请退款 → 通知用户 | 需要调用业务系统 API |
| **数据分析** | 取数 → 清洗 → 分析 → 画图 → 解读 | 需要执行 SQL、运行代码、生成图表 |
| **个人助理** | 管日程 → 订会议 → 写邮件 → 设提醒 | 需要操作日历、邮件等多个工具 |
| **自动化运维** | 监控告警 → 定位问题 → 执行修复 → 验证 | 需要执行命令、查日志、改配置 |

:::info 为什么 2024 年后 Agent 爆发
因为 LLM 的能力达到了"能可靠地理解指令、规划步骤、正确调用工具"的门槛。早期的规则引擎做不到真正的"理解"，而现在的大模型让 Agent 的"大脑"够用了。
:::

## 用一个生活案例理解 Agent 工作流程

假设你对 Agent 说：**"帮我研究一下'折叠屏手机'值不值得买，给我一份建议。"**

Agent 的工作流程是这样的：

```
输入: "帮我研究一下折叠屏手机值不值得买"

【第 1 轮 - 规划】
Agent 思考: 这个任务需要了解价格、优缺点、用户评价
计划: ①搜索折叠屏手机推荐 ②搜索缺点和吐槽 ③对比价格 ④总结建议

【第 2 轮 - 行动】
Agent 决定调用: searchWeb("2024 折叠屏手机 优缺点")

【第 3 轮 - 观察】
工具返回: "优点：大屏体验好、科技感强；缺点：折痕明显、价格贵、维修成本高..."

【第 4 轮 - 行动】
Agent 决定调用: searchWeb("折叠屏手机 用户真实评价 缺点")

【第 5 轮 - 观察】
工具返回: "续航一般、App 适配不全、铰链耐用性存疑..."

【第 6 轮 - 记忆 & 整合】
Agent 把两次搜索结果整合，记下关键信息

【第 7 轮 - 输出】
Agent 回答: "基于调研，折叠屏手机适合追求新鲜感和多任务的用户，
但存在折痕、续航、适配等问题，价格普遍 8000+，建议预算充足且能接受
妥协的用户考虑，否则建议等下一代更成熟的产品。"
```

这个 **"规划 → 行动 → 观察 → 再规划 → … → 输出"** 的循环，就是 Agent 最经典的工作模式，叫做 **ReAct**（Reasoning + Acting）。后面的架构章节会详细讲。

## 当前主流 Agent 框架概览

你不需要从零造轮子，社区已经有成熟的框架。先有个全局认识：

| 框架 | 语言 | 特点 | 适合场景 |
|------|------|------|---------|
| **LangChain / LangGraph** | Python / JS | 生态最大，组件丰富；LangGraph 专注工作流图 | 通用 Agent、复杂工作流 |
| **CrewAI** | Python | 多 Agent 协作友好，角色+任务模型直观 | 多 Agent 团队协作 |
| **AutoGen**（微软） | Python | 多 Agent 对话，支持人在回路 | 研究、代码生成 |
| **Dify** | 多语言 | 低代码平台，可视化搭建 | 快速搭建、非程序员 |
| **OpenAI Assistants API** | 多语言 | 官方托管，内置工具和检索 | 快速集成、不想自建 |
| **Vercel AI SDK** | TS/JS | 前端友好，流式输出好 | Web 应用集成 Agent |

:::tip 给前端同学的建议
- 想快速上手且要前端集成 → **Vercel AI SDK** + **LangChain.js**
- 想理解原理、手写 Agent → 先用 **OpenAI SDK 裸写**（本手册的实战章节就是这么做的）
- 想做多 Agent → 了解 **CrewAI / LangGraph** 的思路
:::

## 本手册的定位

本手册**不绑定某一个框架**，而是从原理出发：

1. 先用**原生 OpenAI SDK** 手写 Agent 核心循环（理解原理）
2. 再介绍 **LangGraph / CrewAI** 的设计思想（学会用框架）
3. 最后**综合实战**构建一个完整项目（融会贯通）

因为框架会变，但 **"感知→规划→行动→记忆→反思"** 的原理不会变。理解了原理，任何框架你都能快速上手。

## 小结

- **AI Agent** = LLM 大脑 + 工具双手 + 记忆 + 规划能力，能自主完成多步任务
- 和 Chatbot 的本质区别：**会"做"而不只是"说"，有自主性而不只是应答**
- 四大核心特征：**自主性、工具使用、规划能力、记忆能力**
- 经典工作模式是 **ReAct 循环**：思考 → 行动 → 观察 → 重复直到完成
- 主流框架很多，但**原理相通**，先学原理再用框架
