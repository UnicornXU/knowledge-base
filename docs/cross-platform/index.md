# 跨端方案

> **"一套代码，多端运行"** —— 跨端开发是前端工程师进阶的重要方向，理解不同方案的原理与取舍是面试高频考点。

## 为什么需要跨端开发？

```
多端开发的困境
═══════════════════════════════════════════════════════

  iOS 原生          Android 原生        Web/H5          小程序
  (Swift/ObjC)     (Kotlin/Java)      (HTML/CSS/JS)   (WXML/WXSS/JS)
      │                  │                 │                │
      └──────────────────┴────────┬────────┴────────────────┘
                                  │
                          开发成本 × N
                          维护成本 × N
                          人才成本 × N

跨端方案的核心目标：
  ✅ 降低开发成本（一套代码 → 多端运行）
  ✅ 统一业务逻辑（减少平台差异导致的 Bug）
  ✅ 提升迭代效率（同步发布、统一维护）
  ✅ 降低团队门槛（前端工程师 → 多端开发）
```

## 跨端方案全景图

```
跨端技术方案分类
═══════════════════════════════════════════════════════

├── Web 技术方案
│   ├── H5 / PWA（纯 Web，最轻量）
│   ├── Hybrid App（WebView + Native Bridge）
│   └── 小程序（双线程，平台管控）
│
├── 原生渲染方案
│   ├── React Native（JS 引擎 + 原生组件）
│   ├── Flutter（自绘引擎，Dart 语言）
│   └── Weex（阿里，已式微）
│
├── 编译转译方案
│   ├── Taro（React 语法 → 多端编译）
│   ├── uni-app（Vue 语法 → 多端编译）
│   └── Kotlin Multiplatform（共享业务逻辑）
│
└── 新兴方案
    ├── Lynx（字节，Web 语法 → 原生渲染）
    ├── Compose Multiplatform（JetBrains）
    └── ArkUI / HarmonyOS（华为鸿蒙）
```

## 方案对比总览

```
跨端方案核心对比
═══════════════════════════════════════════════════════

                React Native    Flutter       Taro/uni-app
                ─────────────  ──────────    ─────────────
  语言           JS/TS          Dart          JS/TS (React/Vue)
  渲染方式       原生组件        自绘引擎       编译为小程序/H5/RN
  性能           ★★★★          ★★★★★        ★★★ (小程序 ★★★★)
  开发体验       ★★★★          ★★★★         ★★★★★
  生态           ★★★★★         ★★★★         ★★★★
  热更新         ✅ CodePush     ✅ 较复杂      ✅ 小程序天然支持
  包体积         中等            较大           小（小程序限制）
  平台一致性     ★★★            ★★★★★       ★★★★
  学习成本       低（前端友好）   中（需学 Dart） 低（前端友好）

选型建议：
  • 已有 React 团队 + 需要原生体验 → React Native
  • 追求极致性能 + UI 一致性 → Flutter
  • 以小程序为主 + 需要多端覆盖 → Taro / uni-app
  • 快速验证 + 轻量需求 → H5 / PWA
```

## 面试高频问题

> 跨端方案是高级前端面试的必考领域。面试官关注你对**渲染原理**、**性能瓶颈**、**方案选型**的理解深度。

**基础题：**
- 跨端方案有哪些类型？各自的优缺点？
- React Native 的新架构（Fabric / TurboModules / JSI）解决了什么问题？
- Flutter 为什么能做到高性能？与 RN 的本质区别是什么？

**进阶题：**
- Taro 3 的运行时方案是怎么工作的？与编译时方案有什么区别？
- 小程序跨端框架的条件编译是如何实现的？
- 如何在跨端项目中处理平台差异？

**方案设计题：**
- 给你一个需要同时覆盖 App + 小程序 + H5 的项目，如何做技术选型？
- 如何设计一个跨端组件库？
