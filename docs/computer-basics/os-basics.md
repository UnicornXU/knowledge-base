# 🔄 操作系统基础

> **"操作系统是计算机的管家"** —— 理解操作系统，才能真正理解程序是如何运行的。

## 一、进程与线程

### 1.1 基本概念

```
进程 vs 线程
═══════════════════════════════════════════════════════

进程（Process）                    线程（Thread）
──────────────                    ──────────────
程序的一次执行实例                 进程内的执行单元
独立的内存空间                    共享进程的内存
资源分配的最小单位                 CPU 调度的最小单位
进程间通信需要 IPC                 线程间可直接通信
创建/销毁开销大                    创建/销毁开销小

关系：
┌─────────────────────────────────────┐
│  进程 A                              │
│  ┌──────────┐  ┌──────────┐         │
│  │  线程 1   │  │  线程 2   │         │
│  └──────────┘  └──────────┘         │
│  共享：堆内存、全局变量、文件描述符    │
│  独立：栈、寄存器、程序计数器         │
└─────────────────────────────────────┘
```

### 1.2 进程的状态

```
进程状态转换
═══════════════════════════════════════════════════════

  创建 ──→ 就绪 ←───────┐
          │  ↑           │
          ↓  │           │
         运行 ──→ 阻塞 ──┘
          │       │
          ↓       ↓
         终止    等待 I/O

就绪 → 运行：被调度器选中
运行 → 就举：时间片用完
运行 → 阻塞：等待 I/O 或事件
阻塞 → 就绪：I/O 完成或事件发生
```

## 二、内存管理

### 2.1 栈内存与堆内存

```
栈内存 vs 堆内存
═══════════════════════════════════════════════════════

栈内存（Stack）                    堆内存（Heap）
──────────────                    ──────────────
存储基本类型                       存储引用类型
大小固定、自动分配                  大小灵活、手动分配/回收
先进后出（LIFO）                   无序存储
访问速度快                         访问速度相对慢
空间较小（通常 1-8MB）             空间较大

内存布局：
┌──────────────────────────────────────┐
│              高地址                    │
│  ┌─────────────────────────────────┐ │
│  │           栈（Stack）            │ │
│  │     局部变量、函数调用帧          │ │
│  │         ↓ 增长方向               │ │
│  ├─────────────────────────────────┤ │
│  │           空闲空间               │ │
│  ├─────────────────────────────────┤ │
│  │         ↑ 增长方向               │ │
│  │           堆（Heap）             │ │
│  │     对象、数组、闭包              │ │
│  └─────────────────────────────────┘ │
│              低地址                    │
└──────────────────────────────────────┘
```

### 2.2 JavaScript 内存管理

```js
// 基本类型 → 栈内存
let a = 10;      // 栈中存储 10
let b = a;       // 栈中复制一份 10
b = 20;
console.log(a);  // 10（不受影响）

// 引用类型 → 堆内存
let obj1 = { name: 'Alice' };  // 堆中存储对象，栈中存储引用地址
let obj2 = obj1;               // 复制的是引用地址
obj2.name = 'Bob';
console.log(obj1.name);  // 'Bob'（共享同一对象）

// 内存分配示例
function createObjects() {
  const arr = new Array(1000).fill(0);  // 堆中分配
  const str = 'hello';                   // 栈中分配（字符串常量）
  const obj = { data: arr };            // 堆中分配
  return obj;
  // arr 和 obj 在函数返回后仍然存活（被引用）
  // str 在函数返回后可被回收（局部变量）
}
```

### 2.3 垃圾回收

```
垃圾回收算法
═══════════════════════════════════════════════════════

1. 标记-清除（Mark-Sweep）
   标记所有可达对象 → 清除未标记对象

   根对象（全局变量、调用栈中的变量）
       ↓
   可达对象（被引用） → 保留
   不可达对象（无引用） → 回收

2. 引用计数（Reference Counting）—— 已淘汰
   每个对象记录被引用次数
   引用为 0 时回收
   ❌ 问题：循环引用导致内存泄漏

3. V8 的分代回收
   ┌─────────────────────────────────┐
   │  新生代（Young Generation）      │
   │  ├── Scavenge 算法               │
   │  ├── 空间小（1-8MB）             │
   │  └── 频繁回收、存活时间短         │
   ├─────────────────────────────────┤
   │  老生代（Old Generation）        │
   │  ├── Mark-Sweep + Mark-Compact   │
   │  ├── 空间大                      │
   │  └── 回收频率低、存活时间长       │
   └─────────────────────────────────┘
```

## 三、进程调度算法

```
常见调度算法
═══════════════════════════════════════════════════════

1. FCFS（先来先服务）
   按到达顺序执行，简单但可能导致"护航效应"

2. SJF（最短作业优先）
   执行时间短的优先，但可能导致长作业饥饿

3. RR（时间片轮转）
   每个进程分配固定时间片，超时则切换
   时间片太大 → 退化为 FCFS
   时间片太小 → 上下文切换开销大

4. 优先级调度
   按优先级执行，可能导致低优先级饥饿
   解决方案：老化（Aging）—— 等待越久优先级越高

5. 多级反馈队列
   结合了 RR 和优先级调度的优点
   新进程进入高优先级队列
   用完时间片则降级
   等待时间长则升级
```

## 四、I/O 模型

### 4.1 五种 I/O 模型

```
I/O 模型对比
═══════════════════════════════════════════════════════

1. 阻塞 I/O（Blocking I/O）
   用户线程 → 系统调用 → 等待数据 → 拷贝数据 → 返回
   线程一直阻塞直到数据就绪

2. 非阻塞 I/O（Non-blocking I/O）
   用户线程 → 系统调用 → 数据未就绪 → 返回错误 → 重试
   需要轮询，CPU 利用率低

3. I/O 多路复用（I/O Multiplexing）
   select/poll/epoll → 监听多个 I/O → 通知就绪的 I/O
   一个线程管理多个 I/O 连接

4. 信号驱动 I/O
   注册信号处理函数 → 数据就绪时收到信号 → 读取数据

5. 异步 I/O（Asynchronous I/O）
   用户线程 → 发起请求 → 继续执行 → 数据就绪后回调
   真正的异步，内核完成所有操作后通知
```

### 4.2 Node.js 的 I/O 模型

```
Node.js I/O 架构
═══════════════════════════════════════════════════════

JavaScript 代码（单线程）
    ↓
 事件循环（Event Loop）
    ↓
 ┌─────────────────────────────┐
 │  libuv（异步 I/O 库）        │
 │  ├── 线程池（Thread Pool）   │ ← DNS、文件 I/O
 │  └── 事件通知（epoll/kqueue）│ ← 网络 I/O
 └─────────────────────────────┘
    ↓
 操作系统内核
```

## 五、死锁

```
死锁的四个必要条件
═══════════════════════════════════════════════════════

1. 互斥条件：资源一次只能被一个进程使用
2. 请求与保持：进程持有资源的同时请求新资源
3. 不可剥夺：已获得的资源不能被强制剥夺
4. 循环等待：进程间形成环形等待链

死锁示例：
  进程 A 持有资源 1，请求资源 2
  进程 B 持有资源 2，请求资源 1
  → 互相等待，永远无法继续

预防策略：
1. 破坏请求与保持：一次性申请所有资源
2. 破坏不可剥夺：允许资源被强制剥夺
3. 破坏循环等待：按顺序申请资源
4. 银行家算法：预判安全状态
```

## 六、前端中的操作系统概念

```js
// 1. 事件循环 —— 类似进程调度
// 浏览器的事件循环调度不同优先级的任务

const worker = new Worker('worker.js');

```

## 七、常见面试题

**Q1: 进程和线程的区别？**

A: 进程是资源分配的最小单位，线程是 CPU 调度的最小单位。一个进程可以包含多个线程，线程共享进程的内存空间。进程间通信需要 IPC，线程间可以直接通信。

**Q2: 为什么 JavaScript 是单线程的？**

A: 避免 DOM 操作的竞态条件。如果多个线程同时修改 DOM，会导致不可预期的结果。单线程 + 事件循环是 JS 的设计选择，通过异步 I/O 和 Web Worker 来处理并发。

**Q3: 浏览器有哪些进程？**

A: 主进程（UI、网络、存储）、渲染进程（JS 执行、渲染）、GPU 进程、网络进程、插件进程。每个标签页通常一个渲染进程（站点隔离下每个站点一个）。
