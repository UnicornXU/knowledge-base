# 🖥️ 计算机基础

> **"基础不牢，地动山摇"** —— 计算机基础是所有技术方向的根基，扎实的基础让你在技术道路上走得更远。

## 为什么前端也需要计算机基础？

```
前端工程师 ≠ 只写页面
    ↓
你需要理解：
┌──────────────────────────────────────────────────┐
│  JS 引擎执行  → 编译原理（词法分析、AST、代码生成）│
│  事件循环      → 操作系统（进程、线程、调度）      │
│  性能优化      → 内存管理（堆栈、垃圾回收）        │
│  代码设计      → 设计模式（单例、观察者、策略）    │
│  数据表示      → 二进制、位运算、浮点数精度        │
│  网络通信      → 操作系统（I/O 模型、系统调用）    │
└──────────────────────────────────────────────────┘
    ↓
计算机基础决定了你的技术天花板
```

**大厂面试越来越重视计算机基础**，尤其是字节、腾讯、美团等公司，操作系统、编译原理、设计模式是高频考点。

## 知识图谱

```
计算机基础
├── 🔄 操作系统          ── 进程与线程、内存管理、调度算法、I/O 模型
├── 📝 编译原理          ── 词法分析、语法分析、AST、代码生成
├── 🏗️ 设计模式          ── 创建型、结构型、行为型、前端常用模式
└── 🔢 数据表示          ── 二进制、位运算、浮点数、编码
```

## 学习建议

1. **操作系统** —— 理解进程/线程是理解 JS 异步的基础
2. **编译原理** —— 理解 AST 是理解 Babel、ESLint、TypeScript 的前提
3. **设计模式** —— 设计模式是写好代码的思维框架
4. **数据表示** —— 理解浮点数精度问题是解决 0.1 + 0.2 的关键

## 面试要点

> 计算机基础在前端面试中通常占 **15-20%** 的比重。操作系统和设计模式是必考内容，编译原理是加分项。建议结合前端实际场景来理解这些概念，而不是孤立地学习。
