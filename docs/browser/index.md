# 🌐 浏览器原理

> **"不了解浏览器，就无法真正理解前端"** —— 浏览器是前端代码运行的宿主环境，理解其工作原理是写出高性能、高质量代码的基础。

## 为什么需要学习浏览器原理？

```
你写的代码
    ↓
浏览器怎么理解它？
    ↓
┌──────────────────────────────────────────────────┐
│  HTML → DOM 树                                    │
│  CSS  → CSSOM 树                                  │
│  JS   → 解析 → 编译 → 执行                        │
│  事件 → 任务队列 → 事件循环                        │
│  内存 → 分配 → 垃圾回收                           │
│  网络 → 请求 → 响应 → 跨域策略                    │
└──────────────────────────────────────────────────┘
    ↓
最终渲染出你看到的页面
```

**面试中浏览器原理的权重极高**，因为它是区分"会用框架"和"理解底层原理"的关键分水岭。

## 学习建议

1. **渲染流程** 是理解性能优化的前提，建议首先学习
2. **事件循环** 是 JS 异步编程的底层机制，面试高频考点
3. **V8 与垃圾回收** 帮助理解内存管理和代码性能
4. **存储与跨域** 是日常开发中解决实际问题的基础
5. **DevTools** 是以上所有知识的实践工具，贯穿始终

## 面试要点

> 浏览器原理在前端面试中通常占 **20-30%** 的比重，尤其是事件循环和渲染流程几乎是必考题。建议通过画图和代码示例来加深理解，而不是死记硬背。
