## 简介
本仓库是一个基于 Docusaurus 的前端面试知识库，重点覆盖 JavaScript、TypeScript、React、Vue、浏览器、网络、工程化与 AI 等专题。其中“手写题”模块聚焦高频面试题，包括防抖/节流、深拷贝、call/apply/bind、new、instanceof 等，配套详尽的思路、实现要点、边界情况与复杂度分析，适合系统化备考与实战演练。

## 详细组件分析

### 节流 throttle
- 目标：在单位时间内只允许函数执行一次，适用于滚动加载、拖拽移动、搜索联想等场景
- 实现范式：
  - 时间戳版：首次立即执行，停止触发后不再执行
  - 定时器版：延迟执行，停止触发后会再执行一次
  - 终极版：支持 leading/trailing 选项与 cancel 方法
- 关键点：
  - this 上下文透传与参数透传
  - 时间精度与内存泄漏防护（cancel）
  - 与 requestAnimationFrame 的结合使用

```mermaid
flowchart TD
Enter(["进入节流函数"]) --> CheckLeading["是否需要首次立即执行?"]
CheckLeading --> |是| ExecNow["立即执行并更新时间戳"]
CheckLeading --> |否| SetLastTime["记录 lastTime"]
ExecNow --> Exit(["返回"])
SetLastTime --> CalcRemaining["计算剩余等待时间"]
CalcRemaining --> CanExec{"剩余时间<=0?"}
CanExec --> |是| ClearTimer["清理定时器(如有)"] --> ExecNow
CanExec --> |否| NeedTrailing{"需要末次执行?"}
NeedTrailing --> |是| Schedule["设置定时器延迟执行"] --> Exit
NeedTrailing --> |否| Exit
```

### 深拷贝 deepClone
- 目标：对对象进行深度复制，支持基本类型、普通对象/数组、Date/RegExp/Map/Set 以及循环引用
- 关键点：
  - WeakMap 缓存已拷贝对象，避免无限递归
  - 通过 Object.getPrototypeOf/Object.create 保持原型链
  - 遍历自身属性（含 Symbol），必要时处理不可枚举属性与 getter/setter
  - 特殊类型构造新实例并拷贝必要状态（如 RegExp.lastIndex）

```mermaid
flowchart TD
Entry(["进入 deepClone(source, hash)"]) --> IsBasic{"是否为基本类型或 null?"}
IsBasic --> |是| ReturnSource["直接返回 source"]
IsBasic --> |否| HasInHash{"hash 中是否已有 source?"}
HasInHash --> |是| ReturnCached["返回缓存副本"]
HasInHash --> |否| SpecialType{"是否特殊类型?"}
SpecialType --> |Date/RegExp/Map/Set| CloneSpecial["创建新实例并递归拷贝"] --> MarkHash["写入 hash"] --> RecurseProps["递归拷贝属性"] --> ReturnTarget["返回 target"]
SpecialType --> |否| CreateTarget["创建空对象/数组"] --> MarkHash --> RecurseProps --> ReturnTarget
```

### call/apply/bind
- 目标：手动实现 Function.prototype.call/apply/bind，理解 this 绑定与原型链
- 关键点：
  - call/apply：将函数挂载为对象的临时方法，执行后删除，注意 Symbol 避免冲突
  - bind：返回新函数，合并预设参数与调用参数，正确处理 new 调用与原型链
  - 边界：context 为 null/undefined、基本类型装箱、箭头函数不可改变 this

### new 操作符
- 目标：模拟 new 的行为，理解对象创建、原型链连接与构造函数执行
- 关键点：
  - 使用 Object.create 建立原型链
  - apply 执行构造函数，this 指向新对象
  - 若构造函数返回对象/函数则使用该返回值，否则返回新对象

### instanceof
- 目标：沿原型链查找，判断 Constructor.prototype 是否在实例的原型链上
- 关键点：
  - 使用 Object.getPrototypeOf 迭代原型链至 null
  - 支持 Symbol.hasInstance 自定义行为
  - 处理基本类型、null/undefined 与非函数右侧参数的边界

```mermaid
flowchart TD
Entry(["myInstanceof(instance, Constructor)"]) --> TypeCheck{"instance 是否为对象或函数?"}
TypeCheck --> |否| ReturnFalse["返回 false"]
TypeCheck --> |是| RightSide{"Constructor 是否为函数?"}
RightSide --> |否| ThrowError["抛出 TypeError"]
RightSide --> |是| UseHasInstance{"是否定义 Symbol.hasInstance?"}
UseHasInstance --> |是| CustomCheck["调用 Constructor[Symbol.hasInstance](instance)"] --> ReturnCustom["返回自定义结果"]
UseHasInstance --> |否| GetProto["let proto = Object.getPrototypeOf(instance)"]
GetProto --> Loop{"proto !== null ?"}
Loop --> |是| Match{"proto === Constructor.prototype ?"}
Match --> |是| ReturnTrue["返回 true"]
Match --> |否| Next["proto = Object.getPrototypeOf(proto)"] --> Loop
Loop --> |否| ReturnFalse
```

## 性能考量
- 节流：优先选择时间戳版本用于“首触即发”的场景；定时器版本适合“末次必达”；复杂场景结合 leading/trailing 与 cancel
- 深拷贝：大对象建议迭代替代递归以避免栈溢出；仅含基本类型的对象可使用 JSON.parse(JSON.stringify()) 优化；结构化数据可用 structuredClone
- call/apply/bind：避免频繁创建闭包与临时属性；在 React 中注意 memo 与稳定引用
- instanceof：原生实现由引擎优化，手写仅作理解用途
